Exercises during class:

  1. Install the Spin model checker. (Or jump to next exercise if it's already installed.)
    • Follow the instructions on http://spinroot.com/spin/Man/README.html.
    • For Windows and Linux, a good start is to copy the executable from http://spinroot.com/spin/Bin/.
    • But you will eventually also need some files from the full distribution, found at http://spinroot.com/spin/Src/.
    • (Note that we will not use iSpin.)
    • For operating systems other than Linux and Windows, you'll probably need to compile the thing from sources.
    • You'll need to have gcc installed for Spin to work. On Linux and other Unixes this will usually not be a problem. (On Ubuntu, you might want to install the build-essential package to avoid future trouble.) For Windows, you might want to have a look at http://www.cygwin.com/ or http://http://www.mingw.org/.
  2. Install the jSPIN graphical interface to Spin. (Or jump to next exercise if it's already installed.)
    • This uses JAVA, so you'll need a JAVA runtime engine. If you don't have this, go to http://www.java.com/.
    • Find jSPIN at http://code.google.com/p/jspin/downloads/. There's a zip file which works fine on Ubuntu; probably on Windows, too.
    • On Linux, I had to make a few changes to the jSPIN config file to make it work; specifically, paths to gcc and help files, and the font used in the tool. I offer you my config file for simplicity.
  3. Start jSPIN and open the example model count.pml which we have talked about. See if you can recognize what I showed you.
  4. Program a Spin model which has a global variable n with initial value 2, and which consists of 2 processes P1 and P2. P1 computes n=n*3 and P2 computes n=n+4. What are the possible values of n at the end?

Homework:

  1. Using the Spin Manual at http://spinroot.com/spin/Man/Manual.html, familiarize yourself with the Spin input language. (Enough to read the first 2 sections of the manual.) Play with jSPIN to see whether you understand the language.
  2. Hand-in exercise: Coming back to the count.pml example: How do the 2 processes need to cooperate to achieve a value of n=2 at the end? Try to find out, using the Spin documentation, how you can get Spin to give you the answer to this question.
