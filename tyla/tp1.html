TP 1 - Rust

Conditions d’achèvement
À rendre : samedi 29 août 2026, 22:42

Goal

You must implement a program handling dictionaries in OCaml 4.14.

A dictionary is a text file containing entries. An entry is one-line long and is composed of a word followed by its definitions. Definitions are separated from the word by a colon : and from themselves by a semicolon ;.

word: definition
word: definition ; definition
word: definition ; definition ; definition
...etc.
  • A word can be any combination of lowercase letters.
  • A definition may be composed of any ASCII character and spans until the end of the line (\n).
  • Dictionaries may contain empty lines.

You must implement a program which takes as argument a path to a dictionary and reads commands on stdin. It must handle the following commands:

  • entries which lists all entries in the dictionary alphabetically.
  • definition <word> which lists the definition of a word alphabetically.
  • synonyms <word> which lists the synonyms of a word alphabetically, i.e., all words which have it as one of their definitions.

Requirements

Cheating (including code copying and hard-coding test results) will result in 0 to this practical as well as a report to the school's administration.

You must provide two files, in a tarball named {login}.tar.gz, similarly to previous practicals:

  • dict.ml which provides the implementation of the Dict module.
  • main.ml which contains the main functionalities of the programs.

The signature for module Dict is:

type t

val empty : t

val add_def : t -> word:string -> def:string -> t
val get_defs : t -> string -> string list
val get_synonyms : t -> string -> string list
val get_entries : t -> string list

I advise you to add this signature to a file dict.mli to ensure that you respect it.

Your code must compile using:

42sh$ ocamlc dict.mli dict.ml main.ml

Note that the order of the files on the command-line matters: dependencies come before the file requiring them.

Some tests (40% of the grade) will be dedicated to your Dict module on its own: keep in mind that you must respect the expected signature for them to compile (and thus succeed).

  • No ill-formed dictionaries or commands will be included in the tests for this practical.
  • Commands tested will always match the dictionary (e.g., a definition command will always match one of the words of the dictionary).
  • You must output results on stdout with the same formatting as the following example. Be wary of not having extraneous whitespaces.

Example

42sh$ ocamlc dict.mli dict.ml main.ml -o tyla
42sh$ cat dict.txt
snake: limbless, elongated reptilian

python: dynamic scripting language ; snake

cobra: snake
anaconda: snake
42sh$ cat input.txt
entries
definition python

definition snake
synonyms snake
42sh$ cat input.txt | ./tyla dict.txt | cat -e
entries:$
- anaconda$
- cobra$
- python$
- snake$
definition python:$
- dynamic scripting language$
- snake$
definition snake:$
- limbless, elongated reptilian$
synonyms snake:$
- anaconda$
- cobra$
- python$
